# PRD — 玉器店进销存管理系统

## 1. 背景

店主经营珠宝玉器实体店，品类涵盖翡翠、水晶、白银、珍珠、檀木、朱砂等（且会持续新增），库存 2000+ 件。主要通过实体门店和微信/朋友圈销售，未来计划拓展电商。

当前痛点（按优先级排序）：
1. 不知道哪些品类利润高、哪些在压货
2. 进货成本和售价记录混乱，算不清楚赚了多少
3. 想看销售趋势但没有数据支撑
4. 拍了大量货品照片但和库存信息对不上

## 2. 用户角色

只有一个角色：**店主**（兼管进货、销售、库存、分析），初期不需要多用户/权限体系。

## 3. 功能需求

### P0 — 核心功能（阶段1-2）

#### 3.1 字典管理
- 店主可新增、编辑、停用材质（一级分类）
- 店主可在某个材质下新增、编辑、停用器型（二级分类）
- 店主可新增、编辑、停用标签（三级，可分组），标签与货品是多对多关系
- 停用的字典项不在录入表单中显示，但已关联的历史数据不受影响
- 初始系统预置常见材质和器型作为种子数据，店主可自行修改

#### 3.2 货品入库
- 支持**单件入库**：填写编号、材质、器型、标签、进价、售价、克重、尺寸、证书编号、供货商、进货日期、备注
- 支持**批量入库**：输入款号(batch_code)、数量、共享进价和售价，系统自动生成 N 个 SKU 编号
- SKU 编号规则建议：`材质缩写-日期-序号`，如 `FC-20240315-001`（翡翠-日期-序号），支持自定义
- 入库后状态为 `in_stock`

#### 3.3 库存查看
- 列表展示所有在库货品，显示：编号、材质、器型、标签、进价、售价、在库天数、缩略图
- 支持按材质、器型、状态、供货商筛选
- 支持按编号、款号、证书编号、备注关键词搜索
- 分页（默认每页 20 条）
- 点击货品可查看详情和所有图片

#### 3.4 销售出库
- 选择一件在库货品，填写实际成交价、销售渠道（门店/微信/电商）、成交日期
- 提交后货品状态自动变为 `sold`
- 已售货品不可重复出库

#### 3.5 借出/归还
- 将在库货品标记为 `lent_out`，记录借出对象和日期
- 借出货品可操作「归还」（回到 in_stock）或「确认售出」（走销售出库流程）

### P1 — 分析功能（阶段3）

#### 3.6 利润看板
- **按材质统计**：各材质的销售额、成本总额、毛利、毛利率、销售件数
- **按渠道统计**：门店 vs 微信 vs 电商的销售额和利润对比
- 支持时间范围筛选（本月/本季/本年/自定义）
- 用图表展示（柱状图或饼图）

#### 3.7 销售趋势
- 按月的销售额和销量折线图
- 可按材质筛选查看单个品类趋势

#### 3.8 压货预警
- 列出在库超过 N 天（默认 90 天，可配置）的货品
- 按库龄降序排列，显示占用资金（进价之和）
- 突出显示高成本长库龄的货品

#### 3.9 图片管理
- 入库时可上传多张货品图片
- 支持设置封面图（列表中显示的缩略图）
- 图片自动生成缩略图（400×400）节省带宽
- 可按货品查看所有关联图片

### P2 — 扩展功能（未来）

#### 3.10 供货商管理
- 记录供货商名称、联系方式、备注
- 可按供货商查看所有进货记录

#### 3.11 数据导出
- 库存列表导出 Excel
- 销售记录导出 Excel
- 利润报表导出 Excel

#### 3.12 条码/二维码
- 给每件货品生成二维码标签
- 扫码快速查看货品信息或出库

## 4. 非功能需求

- **响应式**：店主经常在手机上操作（微信卖货时顺手查库存），前端必须适配移动端
- **速度**：2000 件数据量级，列表加载应在 1 秒内
- **备份友好**：SQLite 单文件 + 图片目录，可直接用极空间的文件同步功能备份
- **离线可用**：部署在本地 NAS，不依赖外部云服务

## 5. 种子数据

系统初次启动时，预置以下常见分类供店主直接使用（可修改/删除/新增）：

**材质**：翡翠、和田玉、水晶、白银、珍珠、檀木、朱砂、蜜蜡、玛瑙、南红

**器型**（通用，各材质可共用）：手镯、挂件、手串/手链、戒指、项链、耳饰、摆件、吊坠

**标签分组**：
- 种水（翡翠用）：玻璃种、冰种、糯冰种、糯种、豆种
- 颜色：满绿、飘花、紫罗兰、黄翡、墨翠、无色
- 工艺：手工雕、机雕、素面
- 题材：观音、佛公、平安扣、如意、山水、花鸟
